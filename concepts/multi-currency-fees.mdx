---
title: Multi-Currency Fees
description: How Magnus lets you pay gas in any stablecoin
---

On every existing blockchain, gas is paid in a single native token — ETH, TRX, SOL, BNB. If you want to send USDT on Tron, you need TRX. If you want to send USDC on Ethereum, you need ETH. You always manage two assets.

Magnus removes this at the protocol level.

## How it works

Magnus transactions include a `feeToken` field — a MIP-20 token address specifying which stablecoin pays the gas fee. The FeeManager precompile handles conversion and settlement automatically.

```
Standard EVM transaction:
  - value: 100 USDT
  - gas paid in: ETH (separate token, must hold both)

Magnus transaction:
  - value: 100 USDT
  - feeToken: mVND_ADDRESS
  - gas paid in: mVND (same wallet, no ETH needed)
```

## The type-0x76 transaction format

Magnus introduces a new transaction type — `0x76` — that extends the standard EIP-1559 envelope with a `feeToken` field.

```typescript
// Standard EIP-1559 (type 0x02)
{
  to, value, data,
  maxFeePerGas, maxPriorityFeePerGas,
  // no feeToken — always ETH
}

// Magnus type-0x76
{
  to, value, data,
  maxFeePerGas, maxPriorityFeePerGas,
  feeToken: '0x...' // any registered MIP-20 token
}
```

The `@magnus/sdk` serializer handles type-0x76 encoding automatically.

## FX reporters

Magnus runs live FX reporters that publish exchange rates on-chain. The FeeManager uses these rates to calculate the correct mVND (or mUSD, mEUR) amount for each transaction's gas cost.

Three FX reporters are active on Staccato devnet:
- mVND/USD rate
- mEUR/USD rate  
- mUSD (pegged 1:1)

## Difference from ERC-4337 paymasters

ERC-4337 paymasters on other chains can sponsor gas for users, but require:
- Smart contract wallet (not standard EOA)
- Developer to run and fund a paymaster service
- Extra transaction overhead per operation
- Only works within one application's scope

Magnus multi-currency fees work for **every transaction, every wallet, every application** — no special setup. It is part of the protocol, not an application-layer workaround.

| | ERC-4337 paymaster | Magnus feeToken |
|---|---|---|
| Works with standard wallets | No (smart wallet only) | Yes |
| Developer setup required | Yes (run paymaster service) | No |
| Universal (all apps) | No (per-app) | Yes |
| Gas overhead | Higher | None |

## Supported fee tokens

Any MIP-20 token registered in the Magnus token registry can be used as a fee token. Current registered tokens on Staccato devnet:

- `mUSD` — Magnus USD (pegged to US Dollar)
- `mVND` — Magnus Vietnamese Dong
- `mEUR` — Magnus Euro

Additional tokens (USDT, USDC when bridged via MBS) are added to the registry as integrations go live.
